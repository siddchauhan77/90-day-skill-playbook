Why this skill

Web development is the foundational skill of the internet economy. Every startup, creator, and small business needs a web presence — and most of them can't build it themselves. The demand is enormous and growing. More importantly, web development is the skill most directly tied to building equity: you can ship your own products, charge for your own SaaS, and own recurring revenue.

The 90-day path focuses on no-code tools (Webflow, Framer) alongside HTML/CSS basics — so you can ship real client work fast while building toward deeper skills over time.

How to sell this skill

The pitch, the proof, what you hand over, and what the client actually measures.

Your pitch

"I'll build you a site that loads under 2 seconds, looks great on mobile, and you can update yourself — delivered in [X] weeks, fixed price."

Proof that closes
  • 3 live URLs with Lighthouse performance scores
  • GitHub profile with consistent commit history
  • 1 case study: before/after speed or conversion data
  • Webflow/Framer template you built and published
What you deliver
  • Deployed site or app (live URL)
  • GitHub repo with clean commit history
  • CMS access + training (if applicable)
  • Style guide / component library
  • Basic docs: how to update, deploy, and maintain
Outcomes to promise
Only things you control
  • Site delivered within the agreed timeline
  • Fully responsive: mobile, tablet, desktop
  • Lighthouse performance score ≥85 on handover
  • Source code + documentation handed over
  • 30-day bug fix window after launch

Fastest path to first client

No cold blast. No waiting 60 days. Here's the exact sequence to land client #1.

First client in ~14–21 days
  1. Day 1: Tell everyone you know: "I'm building websites for small businesses. Know anyone who needs one?" Text, DM, post on socials. Your network is faster than any job board.
  2. Days 2–5: Find 3 local businesses with a broken, dated, or nonexistent website. Screenshot the problems. Pitch a –k rebuild with a 1-week turnaround. Show, don't tell.
  3. Days 6–10: Rebuild your own portfolio or a spec project for a brand you admire. Post the before/after publicly. This is your proof of work before your first paying client.
  4. Days 11–21: Post on r/forhire, r/learnprogramming job threads, and Contra. Respond the same day. Speed of reply signals reliability — that's half the reason clients hire freelancers over agencies.
⚡ Upwork and Fiverr work but are slower. Use them as a secondary channel, not your first move — the fee structure punishes new accounts until you have reviews.

Build for yourself first

Do the thing for yourself before you sell it to others. The output becomes your proof of work.

Build your own portfolio site

Not a template — a custom Webflow or Framer build scored 95+ on Lighthouse. Screenshot the score. That's your first proof.

Proof → Lighthouse score screenshot

Build one free public tool

Solve a real problem for a specific audience (e.g., a copywriter's headline scorer, a freelancer rate calculator). Ship it. Gets inbound.

Proof → Live URL + usage stats

Clone a famous site

Rebuild Stripe's or Linear's homepage from scratch. Post the process. Shows technical range.

Proof → GitHub repo + post

Document a client site rebuild

Even if pro bono — before/after speed score, screenshots, decisions made. One real case study closes clients.

Proof → Case study doc
